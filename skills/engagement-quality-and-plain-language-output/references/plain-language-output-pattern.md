# Plain-Language Output Pattern

Every client-facing finance output uses three layers.

## Layer 1: Business meaning

Explain what happened in operational language:

- Cash received or paid.
- Sales made.
- Stock moved.
- Payroll obligation created.
- Asset consumed.
- Tax or statutory liability created.

## Layer 2: Accounting treatment

State the accounting mechanics:

- Recognition and measurement.
- Journal effect.
- Account and control-account effect.
- Reporting line effect.
- Framework or policy basis.

## Layer 3: Evidence and caveat

State what supports the output:

- Source document.
- Source-register state where statutory values appear.
- Reviewer status.
- Open caveats.
- Action needed before final release.

## Rejection rule

If any layer is missing, the output cannot be labeled final.
